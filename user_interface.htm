Пользовательский интерфейс

 
Произведём запуск программы, с помощью команды – start с точкой, как показано ниже – на рисунке 2.6.
 
 
Рисунок 2.6 – Запуск программы
 
Следующим шагом откроется диалоговое окно взаимодействия с пользователем, первым делом нас встречает приветственный интерфейс – рисунок 2.7.
 
 
Рисунок 2.7 – Приветственный интерфейс
 
Следующим шагом будет уточняющий запрос о имени пользователя. Пользователь системы вводит личные данные, как показано на рисунке 2.8.
 
 
Рисунок 2.8 – Запрос имени
Далее будет предложено выбрать предметную область того вида власти, который по мнению пользователя сможет повлиять на решение его вопроса. Выбор типа номера продемонстрирован на рисунке 2.9.
 
 
Рисунок 2.9 – Запрос предметной области
 
После этого, как показано на рисунке 2.10 будет непосредственный выбор органа власти. Запрашиваемые данные берутся на основе фактов, указанных в коде, исполнительная власть – это правительство и МВД, законодательная – гос. Дума и Совет Федерации, судебная – Верховный и Конституционный суд.
 
 
Рисунок 2.10 – Выбор органа власти
 
Следующим шагом идут вопросы о региональной принадлежности пользователя. На рисунке 2.11 – о городе и области, по которым территориальные органы будут готовы произвести приём граждан.
 
 
Рисунок 2.11 – Запрос территории
 
В конечном итоге, запрашивается дата по маске ввода, как показано на рисунке 2.12. Именно на эту дату записывается пользователь на приём, расчет даты происходит на основе введённой даты, с сопутствующими условиями.
 
 
Рисунок 2.12 – Запросы приёма
 
Далее программа выводит данные пользователя в виде строк, как показано на рисунке 2.13.
 
 
Рисунок 2.13 – Вывод номеров
 
Кроме того, программа выводит введённые параметры в консоли, по которым, в случае необходимости, разработчик сможет отслеживать ошибки программы и вносить соответствующие изменения – рисунок 2.14.
 
 
Рисунок 2.14 – Консольный вывод
 
В следующих пунктах опишем возможные проверки. Во-первых, пользователь может ввести пустые строки. Когда пользователь сознательно или случайно выбирает пропустить ввод требуемой ошибки, система предупреждает его об этом сообщением об ошибке ввода, – рисунок 2.15.
 
 
Рисунок 2.15 – Ошибка ввода
 
Когда пользователь вводит дату, то она должна быть с проверкой корректности ввода по маске: 2025-06-15, и в случае нарушения информировать пользователя.
 
 
Рисунок 2.16 – Ошибка даты
 
В результате создаётся система с простой, но продуманной логикой пользовательского интерфейса, гибкой в плане расширения и адаптированной к русскоязычной аудитории. Она может быть основой для административных сервисов, образовательных проектов или навигации по структурам государственной власти.
